# Roadmap

这个路线图用于记录长期演进方向，不代表必须一次性完成。

## Near Term

- 持续补充真实研发场景中的 Agent、Prompt、Workflow 和 Checklist。
- 为高频场景补充最小可用示例。
- 记录每个 Artifact 的失败案例和限制。

## Possible Future Areas

当真实内容积累足够时，可以考虑增加：

- `artifacts/skills/`：可安装或可复用的 Skill。
- `artifacts/mcp/`：MCP Server、工具说明和集成经验。
- `artifacts/evaluations/`：Prompt、Workflow、Agent 的评估方法。
- `artifacts/automation/`：脚本、CLI 和自动化工具。
- `artifacts/benchmarks/`：面向研发效率的基准测试。

## Review Questions

定期整理时可以问：

1. 哪些 Artifact 被真实使用过？
2. 哪些 Artifact 只是想法，没有产生价值？
3. 哪些内容可以合并？
4. 哪些流程可以加入更清晰的人工确认点？
5. 哪些经验应该沉淀为模板或 Checklist？
