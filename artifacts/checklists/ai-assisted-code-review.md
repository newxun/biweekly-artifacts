# AI Assisted Code Review Checklist

用于 AI 辅助 Code Review 后的人工复核。

## Change Understanding

- [ ] 我能用几句话说明本次变更的目标。
- [ ] 我知道主要修改了哪些模块。
- [ ] 我知道是否涉及数据库、配置、权限、外部 API 或部署流程。

## Correctness

- [ ] 关键路径逻辑与需求一致。
- [ ] 边界条件被考虑。
- [ ] 错误处理符合项目约定。
- [ ] 没有明显的空值、并发、状态不一致或数据丢失风险。

## Tests

- [ ] 新增或修改的行为有对应测试。
- [ ] 回归风险较高的路径有覆盖。
- [ ] 测试结果已实际运行并记录。
- [ ] 如果没有测试，原因是明确且可接受的。

## Maintainability

- [ ] 代码命名清晰。
- [ ] 复杂逻辑有必要说明。
- [ ] 没有引入不必要的抽象。
- [ ] 没有把一次性逻辑包装成长期接口。

## AI Review Validation

- [ ] AI 的每条发现都有明确代码依据。
- [ ] 已删除误报、泛泛而谈或缺少上下文的评论。
- [ ] AI 没有替代人工业务判断。
- [ ] 最终评论表达清楚、具体、可执行。

## Merge Readiness

- [ ] 阻塞问题已处理。
- [ ] 非阻塞建议已明确标注。
- [ ] 风险和后续计划已记录。
- [ ] 合并决策由开发者完成。
